# 📋 Home+ 要件定義

## 1. プロジェクト概要

Home+ は、日常生活に必要な情報をひとつにまとめるライフダッシュボードです。

買い物メモ、献立、読書記録、ゴミの日、天気など、毎日確認する情報を一元管理し、「毎日開きたくなるホーム画面」を目指して開発します。

本プロジェクトは、Python・Streamlitを利用したWebアプリケーション開発だけでなく、GitHubを活用した開発フローやAWSでのデプロイ・運用まで学ぶことを目的としたポートフォリオプロジェクトでもあります。

---

# 2. 開発目的

本プロジェクトの目的は以下のとおりです。

* 日々の生活を少し便利にするWebアプリを作成する
* Python・Streamlitを利用したWebアプリ開発を学ぶ
* GitHubを利用した実践的な開発フローを経験する
* AWSを利用したクラウド環境へのデプロイ・運用を学ぶ
* 長期的に機能追加しながら育てられるアプリを作る

---

# 3. コンセプト

## 「毎日を少し便利に。」

Home+ は、毎日利用する情報をひとつにまとめることで、生活を少し便利にすることを目的としています。

必要な情報を素早く確認できるホーム画面を中心に、自分自身が毎日使いたくなるアプリを目指します。

新しい機能を追加する際も、

> 「その機能は毎日の生活を少し便利にするか？」

を判断基準として開発を進めます。

# 5. 機能要件

## 🏠 ホーム

ホーム画面では各機能の概要を表示します。

表示内容

* 今日の日付
* 今日の天気
* 今日のゴミの日
* 買い物メモ
* 今日の献立
* 読書中の本

ホーム画面では詳細編集は行わず、各機能ページへの入口として利用します。

---

## 🛒 買い物メモ

# 8. 開発フロー

Home+ では GitHub Flow を採用します。

```
Issue作成
    ↓
要件・設計確認
    ↓
featureブランチ作成
    ↓
実装
    ↓
Commit
    ↓
Push
    ↓
Pull Request
    ↓
Merge
    ↓
Issue Close
```

機能ごとにIssueを作成し、ブランチを分けて開発を進めます。
